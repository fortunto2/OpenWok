## Failure Patterns

### Pattern 1: Deploy Spin-Loop (AskUserQuestion in Pipeline)

- **Occurrences:** 5 iterations (iters 4–8 in Run 1)
- **Root cause:** Deploy skill found no deploy CLIs, no Dockerfile, no fly.toml, no stack YAML for Rust. Called AskUserQuestion in every iteration. Responses don't persist across pipeline iterations.
- **Wasted:** 5 iterations = 31% of total pipeline
- **Circuit breaker evasion:** Each iteration had slightly different output (different tool calls, different question wording), generating different md5 fingerprints. Breaker limit is 3 but counter kept resetting to 1.
- **Progression:**
  - Iters 4–6: Check environment → find nothing → AskUserQuestion ("Куда деплоим?") → exit
  - Iter 7: Created Dockerfile, fly.toml, .dockerignore, pushed 18 commits. No Fly auth.
  - Iter 8: Installed flyctl via Homebrew. Still no auth → AskUserQuestion again
  - Iter 9: Attempted `fly auth login`. Wrote "Skipped" state file to exit loop.
- **Status:** PARTIALLY FIXED — Run 9 (pilot-infra) deploy was clean (1 iter, CF Workers), but deploy skill still lacks explicit pipeline mode detection. The success came from wrangler being installed and authed + wrangler.toml already existing.

### Pattern 2: Premature Plan Archival (cf-workers-deploy)

- **Occurrences:** 1 track
- **Root cause:** cf-workers-deploy_20260320 was archived to `docs/plan-done/` with 0/8 criteria met and 0/18 tasks done. Plan status still shows "Not Started". The actual CF Workers deployment was done in Run 9 (pilot-infra deploy stage), not from this plan.
- **Impact:** Misleading — plan in `plan-done/` appears complete but isn't. Future agents could reference it as "done" when the Repository abstraction hasn't been implemented.
- **Fix:** Pipeline should validate plan task completion before archiving. If <50% done, archive to `plan-done/` with `Status: Deferred` or keep in `docs/plan/`.

### Pattern 3: Deploy State File Hack

- **Occurrences:** 1 (iter 9, Run 1)
- **Root cause:** Deploy couldn't deploy (no Fly.io auth), wrote "Skipped" to state file to exit loop
- **Impact:** False positive — deploy marked complete without deployment
- **Status:** Superseded — actual deploy happened in Run 9 via CF Workers

### Pattern 4: Orphaned Fly.io Config

- **Occurrences:** Dockerfile, fly.toml, .dockerignore created in Run 1 for Fly.io, superseded by wrangler.toml in Run 7
- **Status:** Still in repo. cf-workers-deploy plan Task 5.3 addresses this but was never executed.

**Why:** md5 fingerprint evaded by slight wording variations across iterations

```diff
 check_circuit_breaker() {
   local STAGE_ID="$1"
   local OUTFILE="$2"
   local STAGE_RESULT="$3"

   if [[ "$STAGE_RESULT" == "continuing" ]]; then
+    # Check for AskUserQuestion (useless in pipeline mode)
+    if grep -q "AskUserQuestion" "$OUTFILE" 2>/dev/null; then
+      CONSECUTIVE_ASK=$((CONSECUTIVE_ASK + 1))
+      if [[ $CONSECUTIVE_ASK -ge $CIRCUIT_BREAKER_LIMIT ]]; then
+        log_entry "CIRCUIT" "AskUserQuestion in $CONSECUTIVE_ASK consecutive iters — stage needs human input"
+        return 1
+      fi
+    else
+      CONSECUTIVE_ASK=0
+    fi
```

### Patch 3: solo-dev.sh — Plan archival validation

**What:** Check task completion before archiving plan to plan-done/
**Why:** cf-workers-deploy archived at 0% completion — misleading

```diff
 archive_plan() {
   local TRACK_DIR="$1"
   local PLAN_FILE="$TRACK_DIR/plan.md"
+  local TOTAL=$(grep -c '^\- \[' "$PLAN_FILE" 2>/dev/null || echo 0)
+  local DONE=$(grep -c '^\- \[x\]' "$PLAN_FILE" 2>/dev/null || echo 0)
+  if [[ $TOTAL -gt 0 && $DONE -lt $((TOTAL / 2)) ]]; then
+    log_entry "WARN" "Archiving plan with only $DONE/$TOTAL tasks done — marking as Deferred"
+    sed -i '' 's/Status: .*/Status: Deferred/' "$PLAN_FILE"
+  fi
   mv "$TRACK_DIR" "$PROJECT_ROOT/docs/plan-done/"
 }
```
