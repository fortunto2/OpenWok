## Failure Patterns

### Pattern 1: Deploy Spin-Loop (AskUserQuestion in Pipeline)
- **Occurrences:** 5 iterations (iter 4-8)
- **Root cause:** Deploy skill encountered no deploy CLI tools, no Dockerfile, no fly.toml. Instead of making autonomous decisions in pipeline mode, it asked the user via AskUserQuestion in every iteration. Responses don't persist across pipeline iterations, so each iter started from scratch.
- **Wasted:** 5 iterations (56% of total pipeline)
- **Progression:**
  - Iter 4-6: Check → find nothing → AskUserQuestion → exit
  - Iter 7: Created Dockerfile/fly.toml/.dockerignore, pushed commits, but couldn't auth to Fly.io
  - Iter 8: Installed flyctl, couldn't auth
  - Iter 9: Attempted auth, gave up, wrote "Skipped: no deploy needed" to state file
- **Fix:** Deploy skill should detect pipeline mode and make autonomous decisions. If no CLI tools → create deploy configs (Dockerfile, fly.toml) and mark as "ready for manual deploy" without spinning on AskUserQuestion.

### Pattern 2: Spec Criteria Not Updated
- **Occurrences:** 1 (post-build)
- **Root cause:** `/build` skill updated `plan.md` task checkboxes but did not update `spec.md` acceptance criteria. All 10 criteria in spec.md remain `- [ ]` despite all work being complete.
- **Wasted:** 0 iterations (cosmetic, but reduces fidelity tracking)
- **Fix:** `/build` skill should mark spec.md acceptance criteria as `[x]` when corresponding tasks complete.
