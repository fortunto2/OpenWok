## Failure Patterns

### Pattern 1: Deploy Spin-Loop (AskUserQuestion in Pipeline)

- **Occurrences:** 5 iterations (iters 4–8)
- **Root cause:** Deploy skill found no deploy CLIs, no Dockerfile, no fly.toml, no stack YAML for Rust. Instead of making autonomous decisions in `--print` pipeline mode, it called AskUserQuestion in every iteration. Responses don't persist across pipeline iterations — each iter started from scratch.
- **Wasted:** 5 iterations = 45% of total pipeline
- **Progression:**
  - Iters 4–6: Check environment → find nothing → AskUserQuestion ("Куда деплоим?") → exit. Same question 3x with slight variation (evaded fingerprint circuit breaker).
  - Iter 7: Finally autonomous — created Dockerfile, fly.toml, .dockerignore, pushed 18 commits. But couldn't auth to Fly.io.
  - Iter 8: Installed flyctl via Homebrew. Still couldn't auth → AskUserQuestion again.
  - Iter 9: Attempted `fly auth login` (opened browser). Wrote state file "Skipped: local dev only" to exit loop.
- **Fix:** Deploy skill needs pipeline mode detection. When no CLI/auth: create configs → push code → mark "ready for manual deploy" → `<solo:done/>`. Never AskUserQuestion in pipeline mode.

### Pattern 2: Spec Criteria Not Updated (mvp-core)

- **Occurrences:** 1 track (mvp-core spec.md)
- **Root cause:** `/build` updated plan.md task checkboxes but not spec.md acceptance criteria. All 10 criteria remain `- [ ]` despite 100% completion.
- **Wasted:** 0 iterations (cosmetic gap)
- **Note:** phase2-frontend spec.md was correctly updated by review (commit 5adecdb).
- **Fix:** `/build` should mark spec.md criteria `[x]` alongside plan.md tasks.

### Pattern 3: Deploy State File Hack

- **Occurrences:** 1 (iter 9)
- **Root cause:** Deploy couldn't actually deploy (no Fly.io auth), so it wrote "Skipped: local dev only" to the state file to exit the loop. Pipeline treated this as "stage complete" and moved on.
- **Impact:** False positive — deploy marked complete without actual deployment.
- **Fix:** State file validation. Pipeline should check state file content, not just existence. "Skipped" should trigger a warning, not "stage complete".

### Patch 3: solo-dev.sh — AskUserQuestion circuit breaker

**What:** Detect AskUserQuestion calls in iter logs and trip breaker after 3 consecutive
**Why:** Current md5 fingerprint was evaded by slight wording variations

```diff
+# Circuit breaker: AskUserQuestion detection
+ASK_COUNT=$(grep -c "AskUserQuestion" "$ITER_LOG" 2>/dev/null || echo "0")
+if [[ "$ASK_COUNT" -gt 0 ]]; then
+  CONSECUTIVE_ASK=$((CONSECUTIVE_ASK + 1))
+  if [[ "$CONSECUTIVE_ASK" -ge 3 ]]; then
+    log "CIRCUIT" "AskUserQuestion detected in $CONSECUTIVE_ASK consecutive iters — aborting stage"
+    touch "$STATES_DIR/$CURRENT_STAGE"
+    echo "Aborted: interactive question in pipeline mode" > "$STATES_DIR/$CURRENT_STAGE"
+    break
+  fi
+else
+  CONSECUTIVE_ASK=0
+fi
```
